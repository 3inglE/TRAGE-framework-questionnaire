## Part 2. Brief Description of TRACE

TRACE is a framework for designing educational agents in human–AI collaborative education. It helps designers analyze learning tasks, configure appropriate agent roles, embed agents into learning activities, clarify human–AI boundaries, and address governance risks. TRACE contains five stages:

- Task Attribute Analysis
- Role Configuration
- Activity Embedding
- Collaboration Boundary
- Evaluation and Governance

## Part 3. Scenario-Based Reading Task

**Scenario 1. Software Engineering Education — Judgment Boundary**

In software engineering education, students often work on programming tasks, debugging exercises, system design problems, and code review activities. Many tasks are digitally structured and allow immediate feedback, making them suitable for educational agents that provide scaffolding, debugging hints, simulation, or formative feedback.

Using TRACE, an educational agent may support students by explaining programming concepts, identifying possible code errors, suggesting debugging strategies, and prompting reflection on design choices. However, architectural judgment, security trade-offs, and responsibility for final system-level decisions should remain human-led. This scenario represents a judgment boundary.

**Scenario 2. Electrical Training — Safety Boundary**

In electrical training, students need to learn circuit principles, diagnostic procedures, equipment operation, and safety protocols. Some tasks can be practiced through simulation, while real-world operations involve embodied skills and safety-critical consequences.

Using TRACE, an educational agent may support students through fault simulation, safety rehearsal, procedural reminders, and diagnostic practice. However, real equipment operation, safety authorization, and responsibility for high-risk decisions must remain under teacher or institutional supervision. This scenario represents a safety boundary.

**Scenario 3. Illustration Education — Authorship Boundary**

In illustration education, students engage in visual concept development, reference exploration, composition, style experimentation, critique, and creative revision. These tasks are open-ended and closely related to aesthetic judgment and creative authorship.

Using TRACE, an educational agent may support students by expanding reference materials, generating alternative visual directions, offering pre-critique feedback, and prompting reflection on design decisions. However, conceptual direction, aesthetic judgment, and final authorship should remain student-led. This scenario represents an authorship boundary.
